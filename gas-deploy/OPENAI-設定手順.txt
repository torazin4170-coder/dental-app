────────────────────────────────────────
■ このツールで OpenAI が動く機能（2026年時点）
────────────────────────────────────────
  ① 施設FAX日報 … 施設向けコメント本文を AI 作成
  ⑦ 歯科検診票（無料検診）… 家族向け下書きを AI 作成
  日報を Drive に保存する機能 … 付属の QC チェックも OpenAI

  ※ ④⑤ 月次帳票などは、まだルールベース下書き（今後 OpenAI 化予定）
  ※ ②③ なども順次 AI を入れていく想定です

────────────────────────────────────────
ステップ1：OpenAI のアカウントと API キーを用意する
────────────────────────────────────────
  1. ブラウザで https://platform.openai.com/ を開く
  2. ログイン（なければ Sign up でアカウント作成）
  3. 左メニューまたは右上から「API keys」（APIキー）を開く
  4. 「Create new secret key」（新しいシークレットキーを作成）を押す
  5. 名前は何でもよい（例：houmon-shika）
  6. 表示されたキー（sk- で始まる長い文字列）をコピー
     ※ この画面を閉じると二度と見られないことが多いです。メモ帳に貼っておく
  7. 「Billing」（請求）で支払い方法を登録しておく
     ※ 使った分だけ課金されます。無料枠の有無は OpenAI の案内に従ってください

────────────────────────────────────────
ステップ2：Google Apps Script にキーを登録する
────────────────────────────────────────
  1. 訪問歯科カルテ用の Googleスプレッドシート を開く
  2. メニュー「拡張機能」→「Apps Script」をクリック（別タブで開く）
  3. 左の歯車アイコン「プロジェクトの設定」をクリック
  4. 下へスクロールし「スクリプト プロパティ」を探す
  5.「スクリプト プロパティを追加」を押す
  6. 次のように入力して保存：
        プロパティ（名前） : OPENAI_API_KEY
        値                 : ステップ1でコピーした sk-... のキー全体
  7. 画面左上の「保存」アイコン（ディスク）を押す

  ※ GEMINI_API_KEY は入れなくて大丈夫です（使いません）

────────────────────────────────────────
ステップ3：プログラム（Main.gs）を最新にする
────────────────────────────────────────
  1. パソコンの Cursor で「AppsScript-Main-差し替え用.gs」を開く
  2. Ctrl+A → Ctrl+C（すべてコピー）
  3. Apps Script の「Main.gs」を開き、中身をすべて Ctrl+V で置き換え
  4. 保存（Ctrl+S）
  5. 画面上部の関数一覧から testWebAppHtmlBuild を選び「実行」
     → ログに「V6 OK」と出れば OK

  ※ 画面（HTML）も以前の手順どおり gas-deploy の 4 ファイルを貼っていること

────────────────────────────────────────
ステップ4：ウェブアプリを再デプロイする
────────────────────────────────────────
  1. Apps Script 右上「デプロイ」→「デプロイを管理」
  2. 鉛筆（編集）アイコンをクリック
  3.「バージョン」を「新バージョン」に変更
  4.「デプロイ」を押す
  5. スマホ・PCでカルテの URL を開き直す（再読み込み）

────────────────────────────────────────
ステップ5：動作確認（① 施設FAX日報）
────────────────────────────────────────
  1. カルテ →「帳票」タブ
  2.「診療日」と「① FAX日報用の施設」を選ぶ
  3.「① 施設FAX日報」をタップ
  4. しばらくするとプレビューが開く
  5. 画面下に「AIが施設向けコメントを作成しました」と出れば成功
  6. 「OPENAI_API_KEY 未設定」「簡易下書き」と出る場合：
     → ステップ2のキー名の typo、またはステップ4の再デプロイ忘れを確認

────────────────────────────────────────
ステップ6：動作確認（⑦ 無料検診票）任意
────────────────────────────────────────
  1. 帳票タブで患者を選ぶ
  2.「⑦ 歯科検診票」を開き「下書き生成」
  3. エラーなく文章が出れば OpenAI 接続 OK

────────────────────────────────────────
（任意）使う AI モデル名を変えたいとき
────────────────────────────────────────
  スプレッドシートの「settings」シートに次の行を追加（A列=キー、B列=値）：

    openai_model      … 共通の既定（例：gpt-4o-mini）
    fax_openai_model  … ① FAX用（空なら openai_model）
    kentai_openai_model … ⑦ 検診票用
    openai_qc_model   … QC用

  おすすめの最初の値：gpt-4o-mini（速く・安め）
  より丁寧にしたい帳票だけ：gpt-4o などに変更可能

────────────────────────────────────────
トラブル時
────────────────────────────────────────
  ・「429」「insufficient_quota」→ OpenAI の請求・残高を確認
  ・「401」→ APIキーが間違っているか、コピー漏れ
  ・キーを変えたあと → 必ず「新バージョン」で再デプロイ
  ・データ（患者・診療記録）はスプレッドシートに残ります。消えません
