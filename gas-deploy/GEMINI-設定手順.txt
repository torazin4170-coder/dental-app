【訪問歯科カルテ】クラウド AI（Gemini）について

※ 現在のバージョンでは Google Gemini 等のクラウド AI は使用しません。



────────────────────────────────────────

■ GEMINI_API_KEY は不要です

────────────────────────────────────────

  スクリプト プロパティに GEMINI_API_KEY を登録する必要はありません。

  既に登録済みでも、カルテは API を呼び出しません（削除しても構いません）。



────────────────────────────────────────

■ 帳票まわりの動き（AI なし）

────────────────────────────────────────

  ① 施設 FAX 日報 … 表は診療記録から自動。施設向けコメントは手入力。

     「📝 施設コメント簡易下書き」は端末・GAS 内だけのルールベース下書き（外部送信なし）

  ⑦ 歯科検診票 … 各項目を手入力

  日報 Drive 保存 … QC ファイルは「AI QC 無効」の固定文言のみ



  ④⑤⑥ 月次帳票など … もともと API 不使用



────────────────────────────────────────

■ デプロイ後の確認

────────────────────────────────────────

  1. Main.gs（AppsScript-Main-差し替え用.gs）と gas-deploy 4 ファイルを GAS に貼り替え

  2. ウェブアプリを「新バージョン」で再デプロイ

  3. カルテ URL を Ctrl+Shift+R で再読み込み（visitDentalAppBundle_v7 キャッシュ）



  ブラウザの開発者ツール → Network で

  generativelanguage.googleapis.com への通信が無いことを確認できます。



────────────────────────────────────────

■ 旧 settings シートの gemini_* 行について

────────────────────────────────────────

  過去に追加した gemini_model 等の行が残っていても動作に影響しません。

  整理したい場合は settings シートから削除して構いません。
